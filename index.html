  1. Describe difference between HTML and CSS with respect to the roles they take in a web page? (5 point)
    2. Answer:html은 홈페이지에 기본이 되는 토대를 만들고, css는 그 위에 글꼴, 색상 등 여러 스타일을 입힌다. 물론 html로도 이 기능을 할 순 있지만, css로 관리하는 것이 더 편리하다. css는 Cascading Style Sheets의 약자로써 말 그대로 html로 작성된 뼈대에 스타일을 더해준다. external이 많은 양의 파일을 일괄적으로 바꿀 때 편리하다.

  5. When we want to set font style using CSS, we can set generic family and font family. What are the difference between them?(5 point) Which one is recommended to state as a last option to show in a web-page? (5 point)
    6. Answer:
    • generic-family - 모양이 비슷한 글꼴들의 집합의 이름 The name of a generic-family, like "serif", "sans-serif", "cursive", "fantasy", "monospace".
    • family-name - 글꼴 자체의 이름 The name of a font-family, like "times", "courier", "arial", etc.

        • 두개를 혼용해서 쓰지만, 어느 하나를 정하라고 한다면 제네릭을 사용해야한다. 왜냐하면, font-family는 하나의 글꼴만을 선택하지만, generic-family는 그 글꼴이 포함된 집합 전체를 나타내기 때문이다. 이러한 형식으로 사용하면 된다.


  6. When we want to control space of an element, we can use padding and margin. What are the difference between them?(5 point)
    7. Answer:padding과 margin 둘 다 여백을 설정하는 속성이지만, margin은 바깥쪽의 여백을 정해주고, padding은 안쪽의 여백을 지정해준다.

  7. When we set font size using CSS, we can use units of em, %, px. How are they different? (5 point)
    8. Answer: 셋 다 사이즈를 지정해 주지만, pixel은 절대적인 값이 지정되어 있어서 페이지를 확대시키거나 컴퓨터 이외의 다른 디바이스에서 볼 때 사이즈가 변하지 않는다. 이에 반해 em과 %는 유동적으로 사이즈가 조절된다. em 과 %의 차이는 거의 없어보이지만, 일부 브라우저에서 텍스트 크기 자체를 조절했을 때, em의 크기 편차가 심히다. 그래서 CSS의 기본글꼴은 %로 설정하는 게 좋다.

  8. There are four ways to set conditional property for a sequence of elements (parent-child, siblings) - we called it as Combinator in the class. Describe the name of the four combinators with brief descripton and give a short example (10 point)
    9. Answer:
    • Descendant selector: 포함관계를 가지는 구조에서 하위에 있는 모든 요소에 스타일을 적용한다.
    • Child selector: 하위에 있는 요소중에 부모 바로 아래의 자식요소에만 적용된다.
    • Adjacent sibling: 같은 계층에 있는 요소중에서 뒤따르는 요소를 선택한다.
    • General sibling: 같은 계층에 있는 요소중에서 조건을 충족하는 모든 동생요소에 스타일을 적용한다.
